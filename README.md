# 🧷 스터디 규칙

## 📌 문제 풀이

- 매주 총 5문제 풀이
- 문제 난이도는 하위 레벨 (Lv.0 ~ Lv.2) 위주로 선정
- 문제는 개념 발표자를 제외한 팀원들이 각 1문제씩 선정
- 매주 스터디장이 `[WeekXX] 문제 리스트` Issue를 생성
- 팀원들은 해당 Issue 댓글에 본인이 선정한 문제를 작성
- 최종 문제 목록은 해당 Issue에서 관리
- 풀이 코드는 개인 폴더에 업로드  
  `(GithubID/주차/문제이름/문제.java, 문제.js, 문제.py)`
- **화요일 자정까지 PR 제출**
- 모든 PR에는 **최소 1개 이상의 코드 리뷰 필수**

### 댓글 양식
- 난이도: Lv.2
- 문제명: 타겟 넘버
- 링크: https://school.programmers.co.kr/

## 📌 설명 방식

- 문제 풀이 후, 자신의 코드 설명 및 느낀 점을 PR에 작성
- 작성된 PR에는 다른 팀원들이 최소 1개 이상 댓글 작성
- 코드 리뷰를 통해 서로 다른 접근 방식과 코드 스타일을 공유
- 발표자는 준비한 이론을 스터디 시간에 함께 풀이하며 설명

## 📌 참여 방식

- 본 저장소를 **Fork** 한다.
- 자신의 Github ID 폴더에 풀이를 업로드한다.
- Commit Message 규칙을 준수하여 커밋한다.
- 작업 완료 후, 원본 저장소로 **Pull Request(PR)** 를 생성한다.
- 다른 팀원의 PR을 확인하고 코드 리뷰를 진행한다.
- 최소 1명 이상의 승인 후 Merge한다.


## 📁 Repository 폴더 구조

```
{GithubID}/week_01/문제이름/solution.{java|js|py}
```

💡 예시:

```
WhiteBin-bin/week_01/K번째수/solution.java
Anchovia/week_01/타겟넘버/solution.py
sooloin/week_01/문자열압축/solution.js
```


## 📝 Commit Message 규칙

```
[PGS] 문제이름 / 이름
```

💡 예시:

```
[PGS] K번째 수 / 백현빈
[PGS] 타겟 넘버 / 전병훈
```
